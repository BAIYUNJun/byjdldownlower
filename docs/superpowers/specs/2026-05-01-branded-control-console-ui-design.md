# DengLin 下载工具品牌化控制台 UI 设计

## 背景

当前 PyQt6 桌面应用已经具备完整的四步下载流程：输入 SFTP 凭据、选择发布配置、选择下载内容、执行下载。现有 UI 是传统顶部步骤条加页面表单的向导结构，功能清楚，但视觉上更像原型：品牌感弱、样式分散、固定尺寸控件较多，长文件名和进度信息在下载页会互相挤压。

本次重设计采用“产品交付控制台”方向：保留现有业务流程和信号流，把界面升级为左侧品牌步骤栏 + 右侧内容工作区的桌面工具体验。目标是让工具更像 DengLin 对外交付的软件，而不是内部脚本的图形包装。

## 目标

- 强化 DengLin 品牌识别：首屏可见品牌、工具名、版本和当前步骤。
- 提升四步流程的可读性：用户随时知道当前在做什么、前后步骤是什么。
- 优化信息层级：把配置、组件、文件、进度、日志分区展示，减少页面拥挤。
- 解决已有 UI 问题：版本下拉加载后未默认选中、长文件名挤压进度条、欢迎页表单间距紧、下载完成速度计时逻辑被重复方法覆盖。
- 收敛样式维护成本：抽取轻量主题和常用组件，减少重复 stylesheet。

## 非目标

- 不修改 SFTP 连接、文件过滤、下载和 worker 线程的业务逻辑。
- 不引入新的 UI 框架或第三方依赖。
- 不改变四步向导的用户任务顺序。
- 不做账号加密存储改造；凭据安全可作为后续单独任务处理。

## 设计方向

### 界面骨架

主窗口改为两栏布局：

- 左侧品牌步骤栏：固定宽度，深色背景，包含 DengLin 品牌、工具名、版本、四个步骤、当前连接/发布摘要。
- 右侧内容工作区：浅色背景，显示当前页面标题、说明、主要表单/列表、底部操作栏。

窗口建议从 `700x560` 提升到约 `900x640` 的默认尺寸，并设置合理最小尺寸。两栏布局能让页面信息有更充足的横向空间，下载页也能避免文件名、进度条、速度挤在一行。

### 步骤状态

左侧步骤使用四态表达：

- 完成：品牌蓝圆点或勾选标记，文本高亮但低于当前态。
- 当前：高亮背景块，显示步骤编号和主标签。
- 未开始：低对比度文本。
- 异常：在对应步骤旁显示错误提示色小标记，用于列表获取失败或下载失败。

步骤文案保持简洁：

1. 连接服务器
2. 发布配置
3. 选择内容
4. 下载任务

### 视觉语言

主色使用品牌蓝系，搭配深色侧栏和浅灰工作区。避免大面积渐变和装饰性图形，重点放在清晰、稳定、可交付的工具气质。

建议颜色：

- 深色侧栏：`#0F172A`
- 主品牌蓝：`#2563EB`
- 蓝色浅底：`#EFF6FF`
- 页面背景：`#F6F8FB`
- 卡片/输入背景：`#FFFFFF`
- 边框：`#D8DEE8`
- 主文本：`#111827`
- 次文本：`#64748B`
- 成功：`#16A34A`
- 错误：`#DC2626`

圆角控制在 6-10px 之间，符合桌面工具质感。卡片只用于选择项、列表容器、任务摘要，不把整个页面层层包成卡片。

## 页面设计

### 1. 连接服务器

右侧工作区标题为“连接服务器”，副标题显示目标 SFTP 地址。表单采用居中但不悬浮的单列布局，宽度约 360-420px：

- 用户名输入框
- 密码输入框
- 记住密码复选框，左对齐到输入框起点
- 主按钮“连接并继续”

表单标签和输入框使用统一间距，避免当前“密码”标签贴近上一个输入框的问题。按钮禁用态保留，但颜色降低对比，不让禁用按钮看起来像可点击主操作。

### 2. 发布配置

右侧工作区标题为“发布配置”。顶部使用 segmented control 切换“标准发布 / 定制发布”：

- 标准发布：显示架构选择、操作系统选择、SDK 版本。
- 定制发布：隐藏架构和操作系统，只显示定制文件夹选择。

架构和操作系统选择卡片统一为同一种选择组件，包含主标签和描述。选中态使用蓝色边框、浅蓝背景和左上状态点。未选中态为白底细边框，hover 只轻微提升边框颜色。

版本下拉加载成功后必须默认选中第一项；如果列表为空，禁用“下一步”并显示空状态文案。刷新按钮保持小按钮样式，和下拉框垂直对齐。

### 3. 选择内容

右侧工作区标题为“选择下载内容”。页面分为主选择区和当前摘要区：

- 标准发布：上方快速选择预设，下方组件复选列表。
- 定制发布：显示定制文件列表，支持滚动，文件名较长时省略显示并保留 tooltip。
- 摘要区：显示当前发布类型、架构、操作系统、版本或定制文件夹，以及已选数量。

预设按钮仍为互斥选择，但样式改为统一的选择卡片。组件复选框保持两列布局，在 Windows/CentOS 限制时禁用对应项，并显示简短原因 tooltip。

### 4. 下载任务

右侧工作区标题为“下载任务”。页面结构改为：

- 文件列表区：显示匹配文件，包含类别标签和文件名；长文件名省略显示，完整文件名放 tooltip。
- 保存目录区：目录输入框 + 浏览按钮。
- 进度区：总进度一行，当前文件名单独一行，单文件进度和速度再单独一行。
- 日志区：深色日志保留，但使用更合适的高度和边距。
- 任务摘要区：显示文件数、保存目录、当前状态、下载速度或完成状态。

当前文件名不再和进度条同一行，避免长文件名压缩进度条。下载完成后按钮状态变为“已完成”，展示“打开文件夹”作为次要操作。

## 代码结构

### 新增 `downloader/ui/theme.py`

集中定义 UI 常量：

- 颜色 token
- 字体大小
- 间距
- 常用 stylesheet 片段
- 主按钮、次按钮、危险/成功按钮样式函数

该文件不持有业务状态，只提供样式和简单 helper。

### 新增 `downloader/ui/components.py`

集中定义复用 UI 组件：

- `PageHeader`：页面标题和副标题。
- `SidebarStepItem`：左侧步骤项。
- `SelectionCardButton`：架构、操作系统、预设共用的可选卡片按钮。
- `SegmentedControl`：标准/定制发布切换。
- `FooterActions`：底部上一步/下一步/开始下载操作栏。
- `ElidedLabel`：长文件名省略显示并设置 tooltip。

组件只处理展示和 Qt 信号，不访问 SFTP 或 worker。

### 修改 `downloader/ui/wizard.py`

`WizardWindow` 负责新的主窗口 shell：

- 创建左侧品牌步骤栏。
- 创建右侧 `QStackedWidget` 工作区。
- 根据当前页更新步骤状态和左侧摘要。
- 保持现有页面切换信号不变。

旧顶部步骤条移除，避免重复导航信号。

### 修改四个页面文件

- `welcome_page.py`：重排连接表单，使用统一按钮样式。
- `config_page.py`：使用统一选择卡片和 segmented control，修复版本默认选中。
- `mode_selection_page.py`：重排预设/组件/摘要，统一禁用态和文件名 tooltip。
- `download_page.py`：重排文件列表、进度、日志、任务摘要，修复重复 `_on_file_completed()`。

### 修改 `downloader/main.py`

全局 stylesheet 缩减为基础字体和原生控件通用样式。页面级和组件级样式尽量由 `theme.py` 与 `components.py` 提供，避免全局样式误伤局部控件。

## 状态与错误处理

- 版本列表加载中：下拉框禁用，状态文案显示“正在获取版本列表...”，刷新按钮禁用。
- 版本列表为空：下拉框禁用，“下一步”不可用，显示“未找到可用版本”。
- 定制文件夹为空：如果无定制文件夹，隐藏发布切换；如果切换后列表为空，显示空状态。
- 文件列表获取失败：保留弹窗，同时在页面内显示错误状态，便于用户不关闭弹窗也能理解当前状态。
- 下载失败：日志区记录错误，任务摘要显示失败状态，主按钮变为“重试下载”。

## 可访问性与可维护性

- 所有主要按钮保持明确文本，不只依赖颜色表达状态。
- 长文件名使用省略号和 tooltip，避免布局抖动。
- 表单控件设置稳定宽高和最小宽度，窗口缩小时保持滚动区域可用。
- 所有可选卡片使用同一组件，减少架构卡片、OS 卡片、预设按钮样式分叉。

## 验证方案

由于项目目前没有自动化测试，先采用语法检查和离屏渲染截图验证：

1. 对修改后的 Python 文件运行 `python3 -m py_compile`。
2. 使用 `QT_QPA_PLATFORM=offscreen` 渲染四个主要页面状态：
   - 连接服务器页
   - 标准发布配置页
   - 标准发布选择内容页
   - 下载任务页
3. 检查截图中：
   - 左侧步骤栏状态正确。
   - 版本下拉框加载后显示第一项。
   - 长文件名不会压缩进度条。
   - 底部按钮不遮挡内容。
   - 标准/定制发布切换时布局不跳出窗口。
4. 如本地环境允许，启动应用进行一次人工点击流程，覆盖返回、刷新、预设选择、目录选择和下载前校验。

## 成功标准

- 应用仍保持原四步业务流程，现有信号传递不破坏。
- 默认窗口下四个页面无明显拥挤、重叠或控件截断。
- 下载页在长文件名场景下进度条宽度稳定。
- 标准发布版本加载后默认选中最新版本。
- 样式重复明显减少，主要颜色和按钮样式集中维护。
